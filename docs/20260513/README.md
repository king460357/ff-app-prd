# 批次 20260513 · FlareFlow 访客模式与游客身份

本批次包含：**访客模式（无 UID、最小化处理）** 与 **同意隐私条款后的游客 UID（未登录全功能）** 的产品需求，服务上市合规与体验统一。

## 关键决策摘要（执行前必读）

1. **访客模式 ≠ 游客 UID**：访客模式下**不分配任何 UID**；仅在用户**同意隐私条款**且**未登录**时分配**游客 UID**。
2. **访客模式采集**：不收集设备号、广告 ID、行为日志（定义以法务最终口径为准）。
3. **播放**：访客模式与游客 UID 均遵循业务规则——**免费集可看，付费集不可看**（付费集解锁走登录后商业化链路，本 PRD 不展开收银台细节）。
4. **游客 UID 能力**：用户同意条款后，**点赞 / My List（收藏与历史）/ 播放进度同步**等对客功能按本 PRD 全开（依赖条款披露范围）。
5. **充值**：仅**已登录**可完成支付；须通过**多处「同意隐私条款」入口**（见 PRD **5.2.13**）使首启拒绝用户仍能取得游客 UID 后再登录充值；**访客模式下 Profile / VIP / 钱包等不得用「仅登录」弹窗替代隐私授权**。
6. **研发直开**：PRD 含 **§1.6** 自检、**§5.0** 能力矩阵、**§3.3** 状态转移、**§8.3.0** 接口意图、**§8.5** SDK 矩阵、**§10** 已拍板默认策略；实现以正文为准，**无需 PM 再澄清业务分叉**（与隐私政策冲突时走法务变更单）。
