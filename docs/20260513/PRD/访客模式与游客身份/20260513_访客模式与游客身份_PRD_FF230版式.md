# FlareFlow App 需求文档

## 访客模式与游客身份

> **说明**：本文档为 **FF 2.3.0 版式**派生稿，便于研发按历史 docx 习惯阅读；**不替代**同目录 `20260513_访客模式与游客身份_PRD.md` 真理源。冲突以真理源为准。

---

## 1. 需求背景

### 1.1 合规背景

为满足上市合规与用户同意、数据最小化要求：用户须对《服务条款》《隐私政策》做明确授权；**未同意前**不得按已同意用户采集、上报行为数据，不得初始化归因/广告等 SDK。

### 1.2 与历史 FF 文档差异

| 历史表述 | 本期工程定义 | 说明 |
|----------|--------------|------|
| 游客 uid / visitor_uid（多指已同意、有服务端标识） | `GUEST` | 已同意隐私、未注册；下发 `guest_session_token` |
| 游客模式 / 未同意浏览（部分版本用 temporary_id） | `VISITOR` | **未同意隐私**；**无**游客票；仅浏览 Discover 等受限能力 |
| 已登录账号 | `LOGGED_IN` | 充值、会员等 |

本期 **不采用** 2.3.0 稿中「temporary_id 与 visitor_uid 切换」作为未同意态主标识；未同意态统一为 **`VISITOR`**，同意后为 **`GUEST`**。

### 1.3 适用范围

| 属性 | 本期范围 |
|------|----------|
| 地域范围 | 出海 App 全量市场（与现网一致；具体 IP/地区策略若有单独 PRD 以该 PRD 为准） |
| 用户范围 | 首次启动未决策隐私用户；已处于 `VISITOR` 再次触发授权入口的用户 |
| 功能范围 | 隐私弹窗 A/B、身份 `VISITOR`/`GUEST`/`LOGGED_IN`、Discover 访客态、入口授权、待解锁页、播放页点按、访客弹层白名单/黑名单、SDK 开关、游客票签发 |
| 非本期 | 收银台 SKU 细节、分享（本期无）、具体 HTTP 接口路径与 DDL |

## 2. 核心定义

### 2.1 隐私合规弹窗

| 弹窗 | 定义 | 出现条件 |
|------|------|----------|
| **弹窗 A**（主授权） | 要求阅读并同意 Privacy Policy + Terms；含链接、必选勾选（若法务要求）、**同意 / 不同意** | 第 3、4 节各触发条件 |
| **弹窗 B**（二次确认） | 用户于 A 点「不同意」后的二次确认；**同意并继续 / 不同意** | **仅当** A 点「不同意」后 |

全 App **仅上述两枚**隐私合规弹窗；不得新增第三套。系统 **通知（推送）权限** 不属于第三枚隐私弹窗。

### 2.2 用户身份

| 用户模式 | 工程枚举 | 定义 | 能力范围（摘要） |
|----------|----------|------|------------------|
| 访客模式 | `VISITOR` | 未同意隐私；无游客 token | Discover 浏览/搜索/免费集；受限 Tab/付费/写账号；运营弹窗关断 |
| 游客模式 | `GUEST` | 已同意隐私、未注册；有游客 token | 对齐现网游客能力 |
| 已登录 | `LOGGED_IN` | 正式账号 | 充值、会员等 |

### 2.3 身份与 token

| 标识 | 生成时机 | 生成规则 | 是否上报行为/广告类数据 | 生命周期 |
|------|----------|----------|-------------------------|----------|
| 无游客 token | `VISITOR` | 不签发 `guest_session_token` | **否**（归因/分析/广告 SDK 不 init） | 直至用户在 A 或 B「同意并继续」变 `GUEST` |
| `guest_session_token` | `GUEST` | 用户于弹窗 A 同意，或 B「同意并继续」 | 依法务与现网游客规则 | 刷新/吊销规则见后端第 6 节 |

### 2.4 弹窗按钮与身份（全局）

| 用户操作 | 关窗 | 身份 |
|----------|------|------|
| A · 同意（勾选等满足法务） | 关 A | `GUEST` |
| A · 不同意 | 出 B | 仍 `VISITOR` |
| B · 不同意 | 关 A+B | `VISITOR` |
| B · 同意并继续 | 关 A+B | `GUEST`（等同 A 同意；**不再回 A**） |

B 上「不同意」须与「同意并继续」同等可点，不得弱化。

## 3. 功能需求

### 3.1 冷启动流程

用户在 App 启动且 **隐私未决策** 时：进入 App 即展示 **弹窗 A**。用户完成第 2.4 节操作后：变为 `GUEST` 则关窗并在 Discover 正常浏览（对齐现网冷启同意后）；仍为 `VISITOR` 则关 A+B 后访客浏览。可反复 A→B 直至在 B 结束本轮。

**图 1** 冷启动隐私弹窗状态流转（实现以研发状态机为准：未决策 → A →（可选）B → VISITOR 或 GUEST）

### 3.2 访客态入口触发隐私弹窗

当身份为 **`VISITOR`** 时，下列入口 **须先弹窗 A**（同一套 UI）：

| 入口 | 补充逻辑 |
|------|----------|
| 点底部 Reels / My List / Profile | 同意前 **底部 Tab 不得切走**，选中态保持 Discover；`GUEST` 后切至所点 Tab |
| 点顶栏「退出访客模式」 | `GUEST` 后顶栏恢复非访客态 |
| 进入待解锁页 | **进店即 A** |
| 播放页点受控控件（见 4.3） | **每次点按**弹 A |

### 3.3 同意 / 拒绝后的页面切换

各入口在关窗后的 **页面 / Tab / 路由** 差异见下表（按钮路径不重复，均遵第 2.4 节）。

| 场景 ID | 触发条件 | 需求逻辑 |
|---------|----------|----------|
| S1 | 冷启，隐私未决策 | 见 3.1 |
| S2 | `VISITOR` 点 Reels / My List / Profile | `GUEST`：关窗并切至所点 Tab；A→B 均在 B 点「不同意」：关 A+B，底部仍 Discover |
| S3 | `VISITOR` 点「退出访客模式」 | `GUEST`：关窗，顶栏非访客态；B「不同意」：仍 `VISITOR` |
| S4 | `VISITOR` 进入待解锁页 | `GUEST`：关 A，留待解锁页按现网；A、B 均「不同意」：默认 pop 上一层，无上一层可留页但禁止白屏/卡死；页内「同意隐私」须同一套 A |
| S5 | `VISITOR` 播放页点受控控件 | `GUEST`：关窗后再点一次原操作；B「不同意」：留播放页；仍 `VISITOR` 时再次点同一控件再弹 A；进待解锁/收银须先 A（及可选 B） |

## 4. 页面展示与能力限制

### 4.1 页面 / 模块展示范围

| 页面/模块 | 访客模式 `VISITOR` | 游客 `GUEST` | 已登录 |
|-----------|-------------------|--------------|--------|
| 底部 Tab Discover | 展示且为选中态 | 展示 | 展示 |
| 底部 Tab Reels / My List / Profile | 可点，先弹 A，同意前不切 Tab | 展示 | 展示 |
| 首页 Discover 内容 | 展示（列表/搜索/按现网起播） | 展示 | 展示 |
| 播放页 | 播放/暂停/返回及手势豁免不弹 A；其余点按弹 A | 现网 | 现网 |
| 待解锁页 | 进店即 A；双不同意可 pop 返回 | 现网 | 现网 |
| 顶栏「退出访客模式」 | 展示（点按弹 A） | 不展示或现网 | 不展示或现网 |

### 4.2 功能项限制对照

| 功能项 | 访客模式 | 游客模式 | 已登录 |
|--------|----------|----------|--------|
| 免费集正片 | 允许 | 允许 | 允许 |
| 付费集正片 | 未解锁不可播；进待解锁触发 S4 | 未解锁不可播 | 按权益 |
| 点赞 / 收藏 / 进度写服务端 | 不允许（播放页先 A） | 允许 | 允许 |
| Top-up / IAP | 不允许 | 不允许 | 允许 |
| Feed/推荐/运营弹层 | 不自动展示（见 4.4） | 现网 | 现网 |
| 登录引导弹层 | 不自动展示 | 现网 | 现网 |

### 4.3 播放页点按规则

| 操作类型 | 是否弹窗 A |
|----------|------------|
| 播放、暂停、返回（含等同系统返回） | 否 |
| 上下滑换集；拖进度/音量/亮度；点空白唤出控制条；纯手势快进 | 否 |
| 其余可点控件（选集、清晰度、倍速、字幕、点进度跳转、点赞、收藏、弹幕、解锁等） | **是，每次点按** |

`GUEST` / `LOGGED_IN`：按现网，不适用上表。

### 4.4 访客态弹层策略

**白名单（允许）**：弹窗 A、B；系统通知（推送）权限及必要短说明。

**黑名单（禁止自动展示）**：登录引导；付费/订阅/各类挽留；增长运营弹层；好评引导；非合规确认弹窗等。播放页运营挽留 **不得遮挡** A/B。Toast 不计入弹窗黑名单。

### 4.5 SDK 初始化策略

| SDK 类型 | `VISITOR` |
|----------|-----------|
| 归因、行为分析、广告联盟、IAP | **不 init** |
| Crash | 仅匿名（无 user id / IDFA / GAID） |

未同意前不得为运营弹窗 **提前 init** 增长/广告依赖。无法细粒度关闭时：延至 `GUEST` 或 `LOGGED_IN` 后首屏再 init。

## 5. 弹窗需求

### 5.1 隐私主授权弹窗（弹窗 A）

**展示时机**

| 时机 | 说明 |
|------|------|
| 冷启 | App 启动且隐私未决策（S1） |
| 访客点 Tab / 退出访客 / 进待解锁 / 播放页受控点按 | 见第 3.2 节 |

**交互逻辑**

| 步骤 | 逻辑 |
|------|------|
| 1 | 展示 A：标题、导语、政策链接、勾选（若需要）、同意/不同意 |
| 2 | 用户点 **同意** 且勾选等满足法务 → 关 A → `GUEST` |
| 3 | 用户点 **不同意** → 展示弹窗 B，身份仍 `VISITOR` |

**文案与结构**

| 项 | 要求 |
|----|------|
| 标题 | 短句，指向隐私与条款 |
| 导语 | 1～2 句；长文进政策页 |
| 链接 | Privacy Policy、Terms of Service 可点 |
| 勾选 | 法务要求时必选；禁止用「登录」替代 consent |
| 按钮 | **同意** / **不同意** |

**文案 Key（`consent_*`）**

| Key | 英文骨架 |
|-----|----------|
| consent_title | Privacy & terms |
| consent_lead | Please read and agree to our Privacy Policy and Terms of Service to use the full app. |
| consent_link_privacy | Privacy Policy |
| consent_link_terms | Terms of Service |
| consent_checkbox_required | I have read and agree to the Privacy Policy and Terms of Service. |
| consent_primary_cta | Agree |
| consent_decline_cta | Disagree |

### 5.2 二次确认弹窗（弹窗 B）

**展示时机**：用户在弹窗 A 点击 **不同意** 后立即展示。

**交互逻辑**

| 用户操作 | 结果 |
|----------|------|
| 不同意 | 关 A+B；`VISITOR`；页面按 S1–S5 对应逻辑 |
| 同意并继续 | 关 A+B；`GUEST`；不回 A |

**文案 Key（`visitor_confirm_*`）**

| Key | 英文骨架 |
|-----|----------|
| visitor_confirm_title | Continue in visitor mode? |
| visitor_confirm_body | If you don’t agree to the Privacy Policy and Terms of Service, you’ll use FlareFlow in visitor mode with basic browsing only. |
| visitor_confirm_primary_cta | Agree and continue |
| visitor_confirm_secondary_cta | Disagree |

### 5.3 埋点（与实现对齐）

| 场景 | 建议 event / 内部 ID | 说明 |
|------|----------------------|------|
| 冷启弹 A | E1 | 与现网实现对齐即可 |
| Reels / My List / Profile | E10 / E11 / E12 | Tab 来源 |
| 退出访客模式 | E13 | 顶栏 |
| 进入待解锁页 | E7 | 进店 |
| 弹窗展示/点击 | 可沿用现网 `compliant_*` 或映射 | action：confirm / refuse；B 二次确认单独事件 |

## 6. 数据与接口

### 6.1 后端能力（无 URL）

同意并签发游客票；刷新/吊销游客票；点赞、我的列表、播放进度；内容与起播权限——研发映射具体接口。

### 6.2 客户端提示（Toast）

| 场景 | 英文示例 |
|------|----------|
| 同意提交失败 | Couldn’t connect. Check your network and try again. |
| 签发游客票失败 | Something went wrong. Please restart the app. |
| 付费集不可播 | This episode is locked. |
| 游客票过期 | Session expired. Tap to agree to Privacy Policy again. |

## 7. 验收要点

| 模块 | 验收项 |
|------|--------|
| 身份 | `VISITOR` 无游客 token；Discover 为底部选中 |
| 弹窗 | 第 2.4 节全入口一致；B「不同意」可点 |
| 场景 | 第 3.3 节 S1–S5 |
| 弹层 | 仅白名单；黑名单不自动弹、不挡 A/B |
| 播放页 | 第 4.3 节 |
| SDK/接口 | 访客不 init 归因/分析/广告；不写账号数据 |
