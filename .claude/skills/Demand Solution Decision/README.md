# decision-table-generator

> 把"用户反馈"快速转化为"产品需求决策表"的 skill。
> A skill that quickly turns user feedback into a structured product decision table.

---

## 简介 / Introduction

`decision-table-generator` 把产品经理在收到用户反馈后的"决策思考过程"封装成 skill，让你能：

- 把模糊的用户吐槽 → 结构化的 16 字段决策表
- 自动判断 P0/P1/P2/P3 优先级
- 自动给出 1-3 个解决方案对比（成本/复杂度/风险）
- 一次输出两份格式：**Claude 研发版**（给研发继续用 AI 编程）+ **飞书版**（直接复制到飞书表格）

适合场景：
- 产品经理梳理需求池
- 客服工单 → 决策建议
- 写 PRD 之前的"上游决策"环节（与 prd-writer 配套使用）

---

## 与 prd-writer 的关系

```
用户反馈
   ↓
[decision-table-generator]  ← 决策"要不要做、怎么做"
   ↓ （决策=处理 且 方案已定）
[prd-writer]                ← 把决策展开成完整 PRD
   ↓
研发 + 设计实现
```

## 输出字段（16 项）

| # | 字段 | 说明 |
|---|------|------|
| 1 | 需求标题 | 一句话定义 |
| 2 | 问题摘要 | 客观化描述 |
| 3 | 建议优先级 | P0/P1/P2/P3 + 理由 |
| 4 | 产品决策建议 | 处理/暂缓/拒绝/合并 |
| 5 | 解决方案对比 | 1-3 个方案 |
| 6 | 最终推荐方案 | 选哪个 + 理由 |
| 7 | 本次做什么 | 实现范围 |
| 8 | 本次不做什么 | 排除项（防扩展） |
| 9 | 用户流程 | Mermaid + 文字 |
| 10 | 页面交互摘要 | 涉及页面/元素 |
| 11 | 验收标准 | 完成 + 红线 |
| 12 | 上线验证指标 | 衡量指标 |
| 13 | 指标计算方式 | 计算公式 |
| 14 | 指标目标值 | 数值或"待产品确认" |
| 15 | 数据来源 | 埋点/日志/任务系统/数据看板 |
| 16 | 是否需要埋点 | 是/否/需研发确认 |

## 安装

将本文件夹放入 `.claude/skills/` 目录下：

```bash
cd /path/to/your/.claude/skills
# 复制 decision-table-generator 文件夹到此处
```

或在 Claude.ai 中作为 Project knowledge 上传 SKILL.md。

---

## 使用示例

**输入**：
> 用户反馈："App 的搜索太智障了，搜'红色连衣裙'结果一堆都不是红色的。"

**输出**：
- 一份给研发用的 YAML 格式决策表
- 一份可直接复制到飞书"产品需求决策表"的 Markdown 决策表

详见 `references/examples.md`。

## 红线原则

- ❌ 不扩展用户未提出的功能
- ❌ 不生成接口/数据库/架构方案
- ❌ 不编造数据
- ✅ AI 推断必须标注 `[AI 推断]`
- ✅ 信息不足写"待确认"，不留空
