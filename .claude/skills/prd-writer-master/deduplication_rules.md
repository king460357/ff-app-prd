# PRD 去重规则

> 同一规则只能完整定义一次。这是保证 PRD 可读性、可维护性的核心原则。

---

## 1. 唯一规则源原则

**核心思想**：同一个业务规则只能完整定义一次，其他位置只引用，不重复描述。

### 示例

| 规则类型 | 应该定义在哪里 |
|---------|---------------|
| 用户分层规则 | "用户分层"章节 |
| 频控规则 | "频控规则"章节 |
| 缓存规则 | "缓存机制"章节 |
| 权限规则 | "权限规则"章节 |
| 比价规则 | "比价机制"章节 |
| 展示逻辑 | "展示逻辑"章节 |

其他章节如果涉及到，就写："**详见第 X 章 X.Y 节**"，不要重复完整描述。

---

## 2. 章节职责分离

每个章节有明确职责，不串味：

| 章节类型 | 职责 | 不该做的 |
|---------|------|---------|
| 规则章节 | 写完整规则 | 不写场景、不写 UI |
| 场景章节 | 写哪些场景适用、场景差异 | 不重复写规则定义 |
| UI 章节 | 写用户看到什么、点什么、状态如何变化 | 不写规则、不写算法 |
| 流程图章节 | 只做图形表达 | 不作为规则定义来源 |
| 验收章节 | 只写如何判断是否达标 | 不重复背景和完整规则 |

---

## 3. 去重输出格式（三段式）

发现重复时，必须按以下格式输出，方便用户**搜索原文直接替换**：

```markdown
### 重复点 N：{重复主题}

**原文：**
> {从用户文档里完整复制的原句或原段落}

**修改后：**
> {给出可直接替换的版本}

**修改原因：**
> {说明为什么重复、应该保留在哪个章节、其他位置应该怎么引用}
```

### 示例

```markdown
### 重复点 1：频控规则在多处重复定义

**原文：**
> （在第 3.2 节）"用户每日最多展示 3 次广告，间隔不少于 5 分钟。"
> （在第 5.4 节）"广告展示需控制频次，每天不超过 3 次，两次之间间隔 ≥ 5 分钟。"
> （在第 6.1 节）"频控逻辑：当日累计展示次数 ≥ 3 或距上次展示 < 5 分钟时不展示。"

**修改后：**
- 保留第 6.1 节作为唯一规则源（最完整）
- 第 3.2 节改为："频控规则详见第 6.1 节"
- 第 5.4 节改为："广告展示受频控约束，规则详见第 6.1 节"

**修改原因：**
> 同一频控规则在 3 个章节重复，且表述略有差异（"不少于" vs "≥"）容易引起研发理解偏差。
> 集中到 6.1 节作为唯一规则源后，未来如调整规则只需改一处，不会出现章节间不一致。
```

## 4. 常见重复类型与处理

### 4.1 流程重复

**症状**：主流程文字描述、Mermaid 流程图、交互说明都写同一套流程。

**处理**：保留流程图作为唯一视觉来源，文字说明只描述关键节点和异常分支。其他地方写"流程详见第 X 章流程图"。

### 4.2 规则重复

**症状**：规则章节和 UI 章节都完整写了规则。

**处理**：UI 章节只写"用户看到什么、操作什么"，规则部分改成"规则详见第 X 章"。

### 4.3 状态重复

**症状**：每个功能模块都重复解释相同的状态（未开始 / 生成中 / 已完成 / 失败 等）。

**处理**：建立统一状态表（放在前置章节），各模块只写"特殊状态"。

### 4.4 验收重复

**症状**：测试用例和验收标准内容几乎一样。

**处理**：合并为"验收与测试"章节，或：
- 测试章节写"**怎么测**"（步骤、数据、断言）
- 验收章节写"**是否达标**"（结果导向的判断）

---

## 5. 去重审查工作流（模式 B 用）

收到 PRD 后：

1. **先扫描高频重复词**：可以建议用户运行 `scripts/check_prd_repetition.py`
2. **逐项对照"常见重复类型"**：流程重复 / 规则重复 / 状态重复 / 验收重复
3. **按三段式输出每个重复点**
4. **给出整体去重建议**：哪些章节应该合并、哪些应该精简

---

## 6. 反例（不要这样做）

❌ **错误示例 1：只说结论不给原文**
> "你这份文档里规则有重复，建议优化一下。"

✅ **正确做法**：必须粘贴原文 + 给出可替换版本 + 说明原因。

❌ **错误示例 2：模糊建议**
> "可以考虑把第 3 章和第 5 章合并。"

✅ **正确做法**：明确指出"3.2 节的频控规则与 5.4 节、6.1 节重复，建议保留 6.1 作为唯一规则源"。

❌ **错误示例 3：替换内容不可复制**
> "把它改成更简洁的形式。"

✅ **正确做法**：直接给出修改后的可替换文本，用户复制粘贴就能用。
