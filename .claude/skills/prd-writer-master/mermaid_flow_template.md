# Mermaid 流程图模板

> 用于"核心用户动线"、"状态机"等需要流程可视化的场景。
> 飞书/Lark 文档支持 Mermaid 渲染（可能需要安装 Mermaid 插件）。

---

## 模板 1：基础流程图（带成功/失败分支）

```mermaid
flowchart TD
    A[开始] --> B[执行动作]
    B --> C{是否成功}
    C -- 是 --> D[成功状态]
    C -- 否 --> E[失败处理]
    E --> F{是否重试}
    F -- 是 --> B
    F -- 否 --> G[结束]
    D --> G
```

---

## 模板 2：核心用户动线（标准 PRD 用）

```mermaid
flowchart LR
    A[用户进入入口] --> B[选择/输入]
    B --> C[提交]
    C --> D{校验}
    D -- 通过 --> E[处理中]
    D -- 不通过 --> F[提示错误]
    F --> B
    E --> G{结果}
    G -- 成功 --> H[展示结果]
    G -- 失败 --> I[失败提示+重试]
    I --> C
```

---

## 模板 3：状态机（任务/流程状态流转）

```mermaid
stateDiagram-v2
    [*] --> 未开始
    未开始 --> 排队中 : 触发任务
    排队中 --> 生成中 : 开始执行
    生成中 --> 已完成 : 执行成功
    生成中 --> 生成失败 : 执行失败
    生成失败 --> 排队中 : 用户重试
    已完成 --> [*]
    生成失败 --> [*] : 用户放弃
```

## 模板 4：用户角色分支（不同角色不同路径）

```mermaid
flowchart TD
    A[用户进入] --> B{用户角色}
    B -- 管理员 --> C[展示完整菜单]
    B -- 普通成员 --> D[展示部分菜单]
    B -- 访客 --> E[展示登录引导]
    C --> F[执行管理操作]
    D --> G[执行普通操作]
    E --> H[跳转登录页]
```

## 模板 5：异常分支详细版

```mermaid
flowchart TD
    A[用户提交] --> B{网络是否正常}
    B -- 是 --> C{校验是否通过}
    B -- 否 --> N[提示网络异常]
    N --> M[显示重试按钮]

    C -- 通过 --> D[发送请求]
    C -- 不通过 --> E[本地提示错误]

    D --> F{服务端响应}
    F -- 成功 --> G[显示成功]
    F -- 失败 --> H{失败类型}
    F -- 超时 --> M

    H -- 业务错误 --> I[显示业务错误文案]
    H -- 系统错误 --> J[显示通用错误+联系客服]
    H -- 权限错误 --> K[跳转登录或提示无权限]
```

## 模板 6：多步骤引导流程

```mermaid
flowchart LR
    Start[开始] --> Step1[步骤 1：填写基础信息]
    Step1 --> Step2[步骤 2：选择类型]
    Step2 --> Step3[步骤 3：确认提交]
    Step3 --> Done[完成]

    Step1 -.返回.-> Start
    Step2 -.返回.-> Step1
    Step3 -.返回.-> Step2
```

## 模板 7：数据同步流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Page as 列表页
    participant API as 后端
    participant Detail as 详情页

    User->>Page: 编辑某条目
    Page->>API: 提交修改
    API-->>Page: 返回成功
    Page->>Detail: 触发同步刷新
    Detail->>API: 重新拉取数据
    API-->>Detail: 返回最新数据
    Detail->>User: 显示最新内容
```

## 绘制要点

### ✅ 推荐做法

1. **节点用名词或简短动作**：`[提交订单]` 而不是 `[用户提交订单到后端处理]`
2. **判断节点用菱形 `{}`**：`{是否登录}`
3. **分支条件简短**：`-- 是 -->` `-- 否 -->`
4. **方向选择**：
   - `TD`（top-down）：垂直流程，适合分支多
   - `LR`（left-right）：水平流程，适合步骤多
5. **覆盖主流程 + 1-2 个关键异常分支**

### ❌ 避免做法

1. ❌ 节点写成长句子（影响渲染）
2. ❌ 在流程图里写规则细节（流程图只做视觉表达，规则细节在文字部分）
3. ❌ 一张图画太多分支（>15 个节点建议拆分多张图）
4. ❌ 不同图的方向不统一（同一份 PRD 内尽量统一 TD 或 LR）
