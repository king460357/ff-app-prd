# 交互细节标准

> 写"功能详细描述"时，按本文档逐项补齐。这是把"模糊需求"变成"可开发需求"的关键。

---

## 1. 按钮：每个按钮必须说明 6 项

| # | 项目 | 说明 |
|---|------|------|
| 1 | 默认是否可点击 | 是 / 否；如否，说明置灰原因 |
| 2 | 点击后是否弹窗 | 是 → 跳转第 2 节；否 → 直接第 3 项 |
| 3 | 点击后页面状态如何变化 | 进入加载中 / 跳转新页 / 局部刷新 / 弹出抽屉 |
| 4 | 是否可重复点击 | 通常加载中应禁用，避免重复提交 |
| 5 | 成功后去哪里 | 留在原页 / 跳转列表 / 跳转详情 / 跳转结果页 |
| 6 | 失败后提示什么 | 提示文案 + 是否保留输入 + 是否可重试 |

### 示例

> **按钮：「保存草稿」**
> 1. 默认状态：必填项填写后才可点击；未填则置灰，提示"请先填写标题"
> 2. 点击后无弹窗
> 3. 点击后按钮文字变为"保存中..."，进入加载中状态
> 4. 加载中不可重复点击
> 5. 成功后：按钮变回"保存草稿"，并在右上角弹轻提示"已保存"
> 6. 失败后：弹轻提示"保存失败，请稍后重试"，输入内容保留，按钮恢复可点击

## 2. 弹窗：每个弹窗必须说明 5 项

| # | 项目 | 说明 |
|---|------|------|
| 1 | 弹窗标题 | 简洁，一句话说明意图 |
| 2 | 弹窗正文 | 描述场景、风险、影响范围 |
| 3 | 按钮文案 | 通常一组（取消 + 确认） |
| 4 | 点击取消的结果 | 关闭弹窗 / 回到原状态 |
| 5 | 点击确认的结果 | 执行操作 + 后续跳转 / 提示 |

### 示例

> **弹窗：删除确认**
> 1. 标题："确定删除该文件吗？"
> 2. 正文："删除后无法恢复，请谨慎操作。"
> 3. 按钮："取消" + "确认删除"（确认按钮使用警告色）
> 4. 取消：关闭弹窗，停留在原列表
> 5. 确认：删除文件，关闭弹窗，列表局部刷新，右上角轻提示"已删除"

### 危险操作弹窗的额外要求

涉及**不可逆操作**（删除、清空、退出、解绑、扣款等）必须：
- 弹窗正文明确说"不可恢复"
- 确认按钮使用警告色（红色等）
- 推荐使用二次确认（输入"删除"两个字）

## 3. 状态：每个功能必须有 6 类状态

| 状态 | 何时出现 | UI 表现 |
|------|---------|---------|
| 空状态（empty） | 数据为空、首次进入、无搜索结果 | 友好图示 + 引导文案 + 引导按钮 |
| 加载中（loading） | 等待请求/计算 | loading 图标 / 骨架屏 / 进度条 |
| 成功（success） | 操作完成 | 结果展示 / 轻提示 / 状态标识 |
| 失败（failed） | 请求失败、规则不满足 | 失败图示 + 错误文案 + 重试按钮 |
| 部分失败（partial_failed） | 批量操作中部分成功部分失败 | 列表标识 + 汇总提示 + 重试入口 |
| 置灰（disabled） | 必要条件不满足 | 按钮置灰 + 鼠标悬停说明禁用原因 |

### 示例：搜索功能的 6 类状态

| 状态 | 表现 |
|------|------|
| 空状态 | 无搜索词时：显示热门搜索；有搜索词但无结果："未找到相关内容，换个关键词试试" |
| 加载中 | 输入后 300ms 防抖 + loading 图标 |
| 成功 | 列表展示结果，每条结果支持点击 |
| 失败 | "搜索失败，请检查网络后重试" + 重试按钮 |
| 部分失败 | 不适用（搜索通常无此状态） |
| 置灰 | 搜索框为空时，搜索按钮置灰，提示"请先输入关键词" |

## 4. 异常处理：必须覆盖 6 类边界

| # | 边界 | 必须说明 |
|---|------|---------|
| 1 | 空内容 | 字段为空、列表为空、未选择 |
| 2 | 超长 | 文本超长、列表过多、文件过大 |
| 3 | 网络异常 | 请求失败、超时、断网 |
| 4 | 无权限 | 未登录、无权限、被踢出 |
| 5 | 并发 | 同时编辑、同时提交、状态冲突 |
| 6 | 格式不符 | 邮箱格式、手机号格式、特殊字符、文件类型 |

### 标准描述格式

```markdown
| 边界场景 | 用户感知 | 系统响应 | 提示文案 |
|---------|---------|---------|---------|
| 标题为空 | 保存按钮置灰 | 不发请求 | （悬停提示）"请先填写标题" |
| 标题超过 100 字 | 输入框边框变红 + 字数计数变红 | 不发请求 | "标题不能超过 100 字" |
| 网络断开 | 加载中超过 10s 后失败 | 自动重试 1 次 | "网络异常，请检查连接后重试" |
| 重复提交 | 第二次点击无响应 | 拦截重复请求 | （无提示，按钮保持加载中） |
```

## 5. 数据规范：每个表单/字段必须说明 6 项

| # | 项目 | 必填示例 |
|---|------|---------|
| 1 | 字段名（中英文） | `title`（标题） |
| 2 | 类型 | string / number / enum / datetime / array |
| 3 | 长度 / 范围 | 1-100 字 / >0 / 0-1000 |
| 4 | 是否必填 | 是 / 否 |
| 5 | 默认值 | 空 / 0 / "未命名" |
| 6 | 校验规则 | 正则 / 枚举范围 / 自定义校验 |

### 标准格式

```markdown
| 字段名 | 中文 | 类型 | 长度 | 必填 | 默认值 | 校验规则 |
|--------|------|------|------|------|--------|---------|
| title | 标题 | string | 1-100 | 是 | 无 | 不能纯空格 |
| email | 邮箱 | string | ≤ 50 | 是 | 无 | 必须符合邮箱格式 |
| age | 年龄 | number | 0-150 | 否 | 无 | 整数 |
| status | 状态 | enum | - | 是 | 未开始 | 枚举：未开始/进行中/已完成 |
```

## 6. 联动规则：跨页面/跨模块的同步

涉及多个页面/模块联动时，必须说明：

1. **数据来源**：本模块的数据来自哪里？
2. **同步时机**：什么时候触发同步？（保存时 / 实时 / 定时）
3. **同步方向**：单向（A → B）还是双向（A ↔ B）？
4. **冲突处理**：如果同时编辑/状态冲突，以哪边为准？
5. **失败回滚**：同步失败时是否回滚？

### 示例

> **场景：列表页编辑某条目，详情页同步刷新**
> - 数据来源：列表页和详情页都从同一接口拉取
> - 同步时机：列表页保存成功后，立即触发详情页刷新（如详情页打开中）
> - 同步方向：列表 → 详情
> - 冲突处理：以列表页最新保存为准
> - 失败回滚：详情页刷新失败时，保留旧数据，右上角提示"数据可能不是最新，建议手动刷新"

## 7. 验收标准：必须可验证、可测试

每条验收标准必须满足：

1. **可观察**：能用眼睛看到、用工具量到
2. **可判定**：能回答"是 / 否"
3. **不留模糊词**：避免"流畅"、"快速"、"美观"等主观词

### ❌ 错误示例

- 页面加载流畅
- 用户体验良好
- 性能优秀

### ✅ 正确示例

- 页面首屏加载时间 ≤ 2 秒
- 表单提交成功后 1 秒内显示成功提示
- 列表支持 1000 条数据滚动不卡顿
- 失败时显示明确错误文案，包含重试按钮
